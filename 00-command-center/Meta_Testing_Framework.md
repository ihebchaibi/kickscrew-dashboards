# Meta Testing Framework

## Purpose

Use this framework when launching or reviewing Meta tests for KICKS CREW ME.

## Campaign Types

| Type | Use For | Main Metric |
| --- | --- | --- |
| Catalog scale | Proven product feed and retargeting/broad purchase intent | ROAS and CPO |
| ASC test | Broad automated scale test | CPO, ROAS, learning stability |
| Manual conversion | Controlled audience/creative testing | CPO, CTR, purchase rate |
| Product angle test | Pokemon, sneakers, cards, seasonal drops | Hook quality and purchase quality |
| Format test | Static, carousel, UGC, short video, collection | CTR, CPO, thumbstop |

## Launch Checklist

- Clear naming: date, market, campaign type, objective, angle.
- Pixel and purchase event confirmed.
- Product set or landing page matches the creative promise.
- Budget fits test risk.
- Creative variants are meaningfully different.
- Exclusions or audience overlap reviewed where relevant.
- UTM structure is consistent.

## Scale Rules

Scale when:

- ROAS is above 7x and CPO is at or below the healthy threshold.
- Purchase volume is enough to trust the result.
- Performance is not from one accidental order or tiny spend.
- Inventory and fulfillment can support more demand.

Scale carefully when:

- ROAS is strong but CPO is above target.
- Spend is still low.
- Attribution gap is above 20%.
- Campaign is still in learning.

## Pause or Fix Rules

Pause or rewrite when:

- ROAS stays below 7x after enough spend.
- CPO is materially above $22 without a clear learning reason.
- CTR is weak and purchase quality is weak.
- Campaign is duplicating another campaign without adding signal.

Fix before pausing when:

- The product set is wrong.
- Creative promise and landing page do not match.
- Budget is too fragmented.
- Tracking or attribution looks broken.

## Review Cadence

- Daily: spend, purchases, CPO, obvious breakage.
- Twice weekly: creative signal, audience overlap, product set quality.
- Weekly: scale/pause decisions, learning summary, next creative backlog.
